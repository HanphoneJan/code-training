# AI 撰写规范

## 任务：撰写算法题目总结文档

### 必须遵循的结构

1. **渐进式讲解**：暴力 → 优化 → 最优
2. **零思考痕迹**：禁止"让我"、"等等"、"实际上"、"修正"等表达
3. **一步到位的示例**：示例推演必须一次性正确，不要先错后对
4. **面向初学者**：每个步骤都要解释"为什么"

### 禁止列表

```
❌ 让我重新检查
❌ 等等，这里出现了
❌ 实际上这个判断有误
❌ 正确的应该是
❌ 仔细检查后发现
❌ 不对，应该是
❌ 修正一下
❌ 重新推演
```

### 代码要求

- 使用 `float('-inf')` 和 `float('inf')` 简化边界处理
- 避免大量 if-else 分支
- 包含类型注解和注释

### 发布前自查

- [ ] 没有思考痕迹？
- [ ] 示例一步到位？
- [ ] 复杂度用表格对比？
- [ ] 代码简洁优雅？

**记住：文档是最终产品，不是草稿。有任何犹豫或修正，在输出前全部删除。**
