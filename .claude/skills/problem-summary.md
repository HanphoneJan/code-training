# 算法题目总结撰写技能

## 使用场景

当需要为 LeetCode 等算法题目撰写教学文档时使用此技能。

## 撰写流程

### Step 1: 理解题目
- 仔细阅读题目描述
- 理解输入输出约束
- 明确时间/空间复杂度要求

### Step 2: 设计解法演进路径

必须按以下顺序组织：

1. **暴力解法**
   - 最容易想到的方案
   - 说明时间/空间复杂度
   - 解释为什么不满足要求（如果有更优要求）

2. **优化解法**（过渡）
   - 利用题目条件（如有序性）
   - 降低复杂度
   - 作为思维桥梁

3. **最优解法**
   - 达到最优复杂度
   - 详细解释核心思想
   - 图解/示例说明

### Step 3: 撰写内容

**必须包含的章节**：
- 题目描述
- 解题思路（分步讲解）
- 完整代码实现
- 示例推演（一步一步）
- 复杂度分析（表格）
- 易错点总结

### Step 4: 质量检查

**删除所有思考痕迹**：
- 搜索并删除："让我"、"等等"、"实际上"、"正确的"、"修正"、"重新"
- 确保示例推演一步到位，没有先错后对
- 确保代码是最终版本，没有注释掉的调试代码

**检查代码质量**：
- 使用 `±inf` 简化边界处理
- 避免冗长的 if-else
- 有清晰的注释

**检查教学效果**：
- 每个关键步骤都有解释
- 使用了通俗的比喻
- 示例具体且完整

## 禁止清单

以下表达绝对禁止出现在最终文档中：

```
❌ 让我重新推演
❌ 等等，这里出现了...
❌ 实际上这个判断有误
❌ 正确的推演如下
❌ 仔细检查后发现
❌ 不对，应该是
❌ 修正一下
❌ 重新检查
```

## 输出示例

### 好的示例推演

```markdown
**示例**：`nums1 = [1, 3]`, `nums2 = [2]`

**初始化**：
- `m=2, n=1`
- `left=0, right=2`

**第一次迭代**：`i=1, j=1`
- `nums1_left = 1`, `nums1_right = 3`
- `nums2_left = 2`, `nums2_right = +∞`
- 检查：`1 <= +∞` ✓，`2 <= 3` ✓
- 两个条件都满足，找到正确划分！

**计算中位数**：
- 总长=3是奇数，返回 `max(1, 2) = 2` ✓
```

### 差的示例推演（禁止）

```markdown
**第一次迭代**：`i=1, j=1`
- 检查：`2 <= 3`？不满足！（错误）
- 所以 `i` 需要增大

等等，这里出现了错误... 让我重新检查：
**正确的第一次迭代**：...
```
